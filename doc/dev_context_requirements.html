1 — Thông tin dev phải có trước khi agent chạy

Bắt buộc

Tech stack cụ thể

Java version, Spring Boot version, DB engine, frontend framework, build tool. Agent cần biết chính xác để generate code đúng.

Bắt buộc

Project structure

Package naming convention, folder layout, layer structure (controller/service/repo). Agent cần đặt file đúng chỗ.

Bắt buộc

Existing code patterns

Upload 2–3 file mẫu (BaseEntity, BaseController, ErrorHandler) để agent học style và follow đúng convention.

Bắt buộc

Dependencies đang dùng

pom.xml / build.gradle hiện tại. Agent biết dùng lib gì đang có, không suggest lib mới không cần thiết.

Rất nên có

ERD / DB schema hiện có

Architect agent dùng để thiết kế phần mới tương thích, tránh duplicate table hay naming conflict.

Rất nên có

API naming convention

REST endpoint pattern: /api/v1/{resource}, response format chuẩn (ApiResponse wrapper), error code scheme.

2 — Format report agent nên gửi về khi hoàn thành task

Summary (cho owner)

1–3 câu tóm tắt: đã làm gì, kết quả gì. Không technical jargon. Ví dụ: "Đã tạo API login/logout với JWT, test coverage 87%".

Deliverables list

Danh sách file/output cụ thể: tên file, loại (code/doc/test), đường dẫn, kích thước/dòng code. Có thể download.

Issues & blockers

Những điểm agent không chắc hoặc cần quyết định của con người. Phân loại: blocking (phải giải quyết trước) vs non-blocking.

Confidence score

0–100% kèm lý do. Thấp vì: requirement mơ hồ / thiếu context / có nhiều cách giải quyết. Giúp owner biết cần review kỹ đâu.

Next steps

Gợi ý bước tiếp theo: "QA Agent có thể bắt đầu", "Cần owner confirm token expiry", "Architect nên review schema trước".

Token & time usage

Tokens consumed, thời gian chạy, số lần retry. Owner theo dõi cost và identify chỗ nào agent đang "looping".

4 — Vaadin views cần implement

Project Dashboard

List projects, status tổng quan, quick actions: new project, view modules.

Project Settings

Form nhập AI config: LLM, tech stack, coding style, active agents, approval mode.

Module Manager

List modules, upload requirement (drag & drop), trigger agent run, view status.

Agent Monitor

Real-time view: agent status, live log stream qua SSE, progress bars.

Report Viewer

Xem report từng agent, deliverables list, download files, approve/reject output.

Dev Context Panel

Upload code mẫu, ERD, convention docs. Agent dùng làm context khi generate.
